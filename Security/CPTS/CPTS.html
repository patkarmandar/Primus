📋 Prerequisites

Basic networking (TCP/IP, ports, protocols), Linux command line, and a learning-first mindset. CCNA-level networking or TryHackMe's Jr. Penetration Tester path are solid on-ramps. No formal certification is required to enroll — the Academy path builds from the ground up.

Recommended preparation timeline
Month 0 · Optional — 2 to 4 weeks
Foundation before the Academy
Complete TryHackMe "Intro to Cybersecurity" and "Jr. Penetration Tester" paths. Consider the eJPT if you are a complete beginner — it sets the mindset. Crucially: set up Obsidian for note-taking from day one. Your personal command reference built now will pay off enormously during the exam.
Months 1–3 · Core Phase
HTB Academy — all 28 modules in order
Work through every module sequentially. Complete every hands-on exercise and skills assessment — they build on each other and 100% completion is mandatory before you can unlock the exam. Take structured notes after each section explaining every command in your own words, not copy-pasted from the module.
Months 3–4 · Practice Phase
External labs and IppSec's CPTS playlist
Watch IppSec's retired HTB machine playlist — it maps directly to the CPTS curriculum. Complete the official HTB CPTS Machine Track (76 machines). Attempt ProLab Dante (14 machines, pivoting-heavy). If budget allows, add Zephyr (AD-focused and closest to the actual exam environment). Reviewers who completed a ProLab universally passed on the first attempt.
Month 4–5 · Capstone Phase
Attacking Enterprise Networks — completely blind
Do the AEN capstone with zero hints and no walkthroughs. Read the letter of engagement, spin up the target, and work through it as a real engagement. Write a complete SysReptor report at the end. This is the closest simulation of the real exam. Afterwards, compare with the guide to fill gaps — then attempt blind again.
Month 5–6 · Exam Phase
Readiness check and sit the exam
If AEN was smooth and needed minimal hints, you are ready. Review your personal cheatsheet, finalize your SysReptor template, and snapshot your Kali VM. Start the exam — you have 10 full days. Multiple passers found their rhythm by day 2 or 3, so do not panic if day 1 feels slow.
Key decisions before you start
💰 Subscription choice

Student plan ($8/mo with a .edu email) covers all Tier I and II modules required for CPTS. Silver Annual (~$490/yr) without a student email. Total over 4 months with a student subscription plus exam voucher is approximately $242 USD — one of the most cost-effective professional pentesting certifications available.

🚫 You cannot skip the path

100% module completion is a hard system gate before you can unlock the exam voucher. Every skills assessment must be submitted and passed. There are no shortcuts — the modules contain knowledge that will be directly tested in the exam environment.

✅ AEN is your dress rehearsal — treat it exactly like the real exam

Multiple first-attempt passers specifically credited going through AEN blind — with a full SysReptor report — as their single biggest preparation move. If you need heavy assistance during AEN, you need more practice before sitting the actual exam. That is the correct signal to read.

📌 Do them in order — no exceptions

Modules build directly on each other. Skipping or jumping around creates invisible knowledge gaps that the exam reliably finds. Every module exists for a reason. The difficulty increases substantially starting around module 20.

You can obtain all 14 flags and still fail with a poorly written report. HTB evaluates report quality rigorously: structure, evidence quality, impact statements, and remediation guidance all count. Core tools to master before exam day
Nmap
Port scanning, service detection, NSE scripts. Know every timing flag and major script category cold.
NetExec (CrackMapExec)
Credential validation, lateral movement across SMB, WinRM, RDP, LDAP. Your primary lateral movement tool throughout the exam.
Ligolo-ng
Encrypted TLS tunneling. Dramatically cleaner than proxychains. Practice multi-hop. Build static binaries from source — don't trust precompiled releases.
BloodHound / SharpHound
AD privilege path mapping. Run SharpHound immediately on any AD foothold. BloodHound visualizes attack paths you would never find manually.
Impacket Suite
Kerberoasting, AS-REP roasting, secretsdump, wmiexec, smbclient, mssqlclient. Know each script and when to use it.
PowerView
AD recon — users, groups, ACLs, trust relationships, delegation settings, GPO enumeration. The AD pentester's best friend.
Rubeus
Kerberos tooling — Kerberoasting, AS-REP roasting, ticket manipulation, S4U2Self/S4U2Proxy delegation abuse, overpass-the-hash.
PowerUp / WinPEAS
Windows local privilege escalation discovery. Service exploits, registry paths, token abuse, scheduled tasks. Run both and compare.
LinPEAS
Automated Linux privilege escalation enumeration. SUID/SGID, writable scripts, sudo misconfigs, capabilities, cron jobs.
Evil-WinRM
PowerShell remoting over WinRM. Your primary interactive shell for Windows post-exploitation and file transfers.
Ffuf
Web fuzzing — directories, parameters, vhosts, file extensions. Know every filtering flag and maintain a curated wordlist collection.
Hashcat + JtR
Offline password cracking. Know hash type numbers (-m values), rulesets (best64, OneRuleToRuleThemAll), and rockyou variants.
Burp Suite Community
Web proxy — repeater, intruder, decoder, comparer. Master the core features cold. Community edition is sufficient for CPTS.
Metasploit / MSFvenom
Staged and stageless payloads, Meterpreter, local exploit suggester. Know what it does — but don't over-rely on it.
Responder
LLMNR/NBT-NS/mDNS poisoning for credential capture. Know when and when not to run it in a given network environment.
Chisel
TCP tunneling over HTTP — useful backup when Ligolo-ng cannot be deployed. Know both client and server modes.
Note-taking — Obsidian setup
📓 Build your personal HackTricks from day one

Use Obsidian with a structured vault pushed to a private GitHub repository. Organize by: Attack Category → Tool → Technique → Command with explanation. Write what each flag does rather than just the command. Sync across devices with Git. Reviewers who built this habit found the exam significantly less stressful — their notes worked as a personal reference under pressure.

⚠️ Writing things down does not equal understanding them

Many candidates copy-paste commands from modules into Obsidian without understanding what the flags do. These notes become useless under exam pressure. Force yourself to explain every technique in your own words — if you can explain it to someone else, you own it. Your notes should be useful to a "future you" with zero context.

Report writing — SysReptor
📄 Set up SysReptor before the exam — use it during AEN first

SysReptor is the community-preferred reporting tool with an official HTB-aligned template. It is Markdown-based, renders to a clean professional PDF, and includes pre-built finding templates matching HTB's expected report structure. Install and use it for your AEN blind run — you do not want to be learning the tool on exam day. GitHub: Syslifters/HackTheBox-Reporting.

Attack platform setup
🖥️ Prepare everything before exam day — snapshot your VM

Install and thoroughly test Ligolo-ng (built from source), NetExec, BloodHound, and all tooling before the exam starts. Take a Kali VM snapshot after setup completes. Set up HTB Pwnbox as a backup attack platform — if your local VM loses VPN connectivity mid-exam, Pwnbox keeps you running immediately without losing momentum.

🏗️ Environment structure
Approximately 8 machines (mix of Linux and Windows), multiple subnets requiring active pivoting, at least one Active Directory domain with trust relationships and privilege chains. You receive a letter of engagement with scope, objectives, and rules — read it carefully and completely before touching anything. The environment simulates a real enterprise network with realistic misconfigurations.
Day-by-day exam strategy from first-attempt passers
📅 Days 1–2: slow down, enumerate everything

Read the letter of engagement before touching any target. Complete full network discovery before attempting exploits. Multiple first-attempt passers obtained zero flags in the first two days — this is completely normal. Document everything from minute one: discovered IPs, open ports, running services, credentials found, every useful command output with clean annotated screenshots.

📅 Days 3–7: build momentum, don't get tunnel vision

Once you find the first foothold the environment opens up. Deploy Ligolo-ng for pivoting, run SharpHound immediately in any AD environment and review results in BloodHound before manual enumeration. Do not spend more than 4–6 hours stuck on a single machine — pivot to another target and return with fresh eyes. Reset the environment if tools behave unexpectedly.

📅 Days 7–9: stop hacking at 12+ flags, start the report

Once you reach 12 flags, shift completely into reporting mode. Return through the environment to capture clean, annotated, high-quality screenshots for each finding. Do not sacrifice report quality chasing the final 1–2 optional flags. The report is where you pass or fail — a complete report with 12 flags beats a sloppy report with 14.

📋 Report requirements — the report IS the exam

Your report must include: an executive summary written for a non-technical C-level reader; an attack path narrative showing how you moved through the environment; individual findings with CVSS severity ratings; full evidence (annotated screenshots for every flag and every key exploitation step); and specific actionable remediation recommendations. Use the exact structure from the Documentation & Reporting module.

Tips from real first-attempt passers
  • 📸
    Screenshot everything the moment you find it. Credentials, shell prompts, flag submission screens, privilege escalation steps — capture all of it immediately. You will deeply regret the missing screenshot when writing the report eight days later.
  • 🔄
    Reset the environment if tools misbehave. Exam labs can become unstable. Do not spend hours debugging a broken environment — a reset takes 15 minutes and frequently resolves mysterious tool or connectivity failures that aren't your fault.
  • 🔌
    Set up Pwnbox as a backup attack platform before day one. If your local Kali VM loses internet or VPN connectivity mid-exam, Pwnbox allows you to resume immediately without losing momentum or wasting exam time.
  • 🗺️
    Diagram the network as you discover it. Use Excalidraw or draw.io to map discovered hosts, subnets, credentials, and pivot chains. Visualizing the complete network prevents routing mistakes and makes lateral movement planning dramatically easier.
  • 😴
    Sleep. The exam is 10 days, not a 48-hour sprint. An exhausted mind misses obvious attack vectors. Multiple reviewers specifically mentioned solving machines they were stuck on for hours after getting a full night of sleep and returning with fresh perspective.
  • 🔧
    Compile Ligolo-ng static binaries from source before the exam. At least one reviewer had precompiled GitHub release binaries fail in the exam environment. Compile from source, test against your practice machines, and bundle the binaries in your tooling before day one.
  • 📝
    Maintain a running notes file throughout the exam. Record every command run, credential discovered, service identified, and anomaly observed. Your exam notes become the evidence base for your report — clean structured notes produce clean structured reports.
  • 🩸
    Run BloodHound immediately in every Active Directory environment. Do not manually enumerate AD relationships before you have a BloodHound graph. Let it map the entire privilege chain visually, then choose the most efficient attack path based on what it reveals.
