The request path becomes:

```text
Browser React app
    -> /api/spaces
Cloudflare Worker you wrote
    -> checks session and ownership
    -> reads/writes D1
    -> returns encrypted payload
Browser decrypts it locally
```

I would put encryption in `packages/crypto` and keep decryption exclusively in the browser. The Worker can store encrypted spaces and items, but should never receive the vault PIN, recovery code, or decrypted note contents.

For deployment, Cloudflare can build `apps/web` as it does now, while a second Cloudflare Worker deployment publishes `apps/api` at `api.archespace.cc` (or `/api` on the same domain). The frontend then replaces `supabase.from(...)` calls with a small `src/api/client.js` wrapper calling your API.
