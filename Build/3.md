Yes. These fall into two different groups:

1. **Backend-as-a-Service (BaaS)**: they give you database, auth, APIs, storage, and often realtime. Fast to build, but their platform remains part of your backend.

2. **Backend hosting/runtime**: you write the API yourself; they just run it. This better matches your goal.

| Service | What it provides | Can you write your own API? | Best fit |
|---|---|---:|---|
| Supabase | Postgres, Auth, APIs, storage, realtime | Yes, but its SDK/API remains central | Fast full-stack apps |
| Firebase | Auth, NoSQL database, storage, functions | Yes | Mobile/realtime apps |
| Appwrite | Open-source auth, DB, storage, functions | Yes; can self-host | Supabase-like alternative |
| PocketBase | Self-hosted Go + SQLite backend in one executable | Custom Go/JS hooks | Small self-hosted apps |
| Convex | TypeScript database functions + realtime | Yes, inside Convex’s model | Reactive TypeScript apps |
| Cloudflare Workers | Runs API code you write | Yes, fully | Arche’s hosted custom backend |
| Vercel Functions | Runs API code you write | Yes | Next.js applications |
| AWS Lambda | Runs API code you write | Yes | Large/enterprise AWS setups |
| Render / Railway / Fly.io | Hosts normal Node/Express servers | Yes, fully | Traditional Express backend |
| VPS (Hetzner, DigitalOcean, etc.) | A server you control | Yes, fully | Fully self-hosted Express + PostgreSQL |

For your specific requirement, the strongest choices are:

- **Cloudflare Workers + Hono + D1**: your custom API, almost no server maintenance, free tier available.
- **Express + PostgreSQL on a VPS**: most traditional and independent; you run everything.
- **PocketBase on a VPS**: simpler self-hosting, but its own backend framework is still part of the architecture. PocketBase itself cautions that it is pre-1.0 and not ideal for production-critical applications unless you can manage migration changes. [PocketBase documentation](https://pocketbase.io/docs/)

I would not move Arche from Supabase to Firebase, Appwrite, or Convex if your reason is “I want my own backend.” That swaps one backend service for another. Cloudflare Workers or Express is the genuine move toward code you control.
