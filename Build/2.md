Yes, both are possible, but they fit different hosting choices.

**Next.js:** Good if you want one project containing the UI and backend routes. You would replace Vite/React with Next.js, then use Route Handlers for API endpoints.

```text
Next.js frontend + API routes
        -> D1/Postgres
        -> Resend
```

On Cloudflare, use the OpenNext Cloudflare adapter. It works, but it is a bigger migration from your current Vite app and has more framework machinery than Arche needs.

**Express.js:** Great for a traditional Node.js server, usually on a VPS, Railway, Render, Fly.io, or similar. But standard Express does not run directly in Cloudflare Workers, because Workers are not Node servers with a permanent listening process.

For Cloudflare Workers, use a Worker-native router such as Hono. It feels very similar to Express:

```ts
app.get("/api/spaces", requireAuth, getSpaces);
app.post("/api/spaces", requireAuth, createSpace);
```

My preference for Arche:

- Keep the existing React + Vite frontend.
- Write your own backend in TypeScript using Hono on Cloudflare Workers.
- Use D1 initially, or PostgreSQL if you expect more complex database needs.

Choose Next.js only if you specifically want its server-rendered pages and one unified web framework. Choose Express only if you want to run and maintain a traditional Node server.
